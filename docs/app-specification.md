# 推し活管理アプリ 仕様書

**バージョン:** 1.0.0  
**作成日:** 2026年7月2日  
**プラットフォーム:** iOS / Android（クロスプラットフォーム）  
**技術スタック:** Flutter (Dart)

---

## 1. アプリ概要

### 1.1 アプリ名（仮）
**Oshi Log（推しログ）**

### 1.2 コンセプト
推しに関するすべての活動（イベント・グッズ・支出・貯金）をひとつのアプリで一元管理する。  
「今月いくら使ったか」「いくら貯められたか」を一目で把握できる、推し活専用の家計簿 × 活動記録アプリ。

### 1.3 ターゲットユーザー
- アイドル・俳優・VTuber・アニメキャラクターなどの推しを持つユーザー
- グッズ購入やイベント参加を記録・管理したいユーザー
- 推しへの出費を把握・コントロールしたいユーザー

---

## 2. 推し管理

### 2.1 推し登録
- グループ単位・個人単位どちらでも登録可能
- グループ登録時、所属メンバーを個別に追加できる
- 各推しに設定できる情報：
  - 名前
  - アイコン画像（ギャラリーから選択 または カメラ撮影）
  - カバーカラー（テーマカラー）
  - 誕生日
  - カテゴリ（アイドル / 俳優 / VTuber / アニメ / その他）
  - メモ（自由記述）

### 2.2 推し一覧
- 登録した推しをカード形式で一覧表示
- タップで推しごとの詳細ページへ遷移
- 並び替え：登録順 / 名前順 / 最近閲覧順

---

## 3. イベント管理

### 3.1 イベント記録
各推しに紐づいてイベントを登録できる。

| 項目 | 内容 |
|---|---|
| イベント名 | 任意のテキスト |
| 日付 | カレンダーから選択 |
| 場所・会場 | テキスト入力 |
| カテゴリ | コンサート / 舞台 / 握手会 / 配信 / その他 |
| 支出金額 | 数値入力（円） |
| チケット代・交通費・宿泊費 | 内訳の個別入力（任意） |
| 写真 | 複数枚添付可能 |
| メモ | 自由記述 |

### 3.2 イベント一覧
- 時系列（新しい順 / 古い順）で表示
- 月でフィルタリング可能
- カレンダービューで参加日を可視化

---

## 4. グッズ・購入履歴管理

### 4.1 グッズ登録
各推しに紐づいてグッズの購入情報を登録できる。

| 項目 | 内容 |
|---|---|
| グッズ名 | テキスト入力 |
| 購入日 | カレンダーから選択 |
| カテゴリ | CD / Blu-ray / 写真集 / アパレル / アクセサリー / 雑貨 / その他 |
| 金額 | 数値入力（円） |
| 購入場所 | テキスト（店舗名 / EC サイト名） |
| 数量 | 数値入力 |
| 写真 | 1〜複数枚添付可能 |
| メモ | 自由記述 |

### 4.2 グッズ一覧
- グリッド / リスト 表示切り替え
- カテゴリでフィルタリング
- 月・期間でフィルタリング

---

## 5. 支出サマリー（ダッシュボード）

### 5.1 月次サマリー
- 当月の総支出額を大きく表示
- 内訳をカテゴリ別に円グラフ / 棒グラフで表示
  - イベント費
  - グッズ費
  - その他
- 推しごとの支出比較（複数推し登録時）

### 5.2 推しごとサマリー
- 選択した推しへの累計投資額
- 月別推移グラフ（折れ線グラフ）
- イベント参加回数・グッズ購入点数の統計

### 5.3 年次サマリー
- 年間の支出推移を月ごとに棒グラフで表示
- 年間総支出額

---

## 6. 推し貯金機能

### 6.1 貯金目標の設定
推しに関連したイベント（誕生日・記念日など）を起点に、日々の貯金を管理する機能。

| 項目 | 内容 |
|---|---|
| 貯金名 | 例：「○○ちゃん誕生日貯金」 |
| 紐づく推し | 登録済みの推しから選択 |
| 開始日 | カレンダーから選択 |
| 目標日（任意） | 目標達成希望日 |
| 目標金額（任意） | 達成したい総額 |
| 1日あたりの貯金額 | 数値入力（円） |

### 6.2 貯金記録
- 毎日の貯金を手動で「今日も貯金した！」ボタンでチェックイン
- 任意で金額を変更して記録可能（その日だけ増額・減額）
- 連続貯金日数（ストリーク）を表示

### 6.3 貯金の可視化
- 月ごとの貯金合計額を表示
- 累計貯金額と目標金額の進捗バー
- カレンダーヒートマップで貯金した日を可視化
- 目標達成時に達成通知 / お祝いアニメーション

### 6.4 複数の貯金を並行管理
- 推しごとに複数の貯金プランを作成可能
- 一覧で各貯金の進捗を確認

## 7. 通知機能

| 通知種別 | 内容 |
|---|---|
| 貯金リマインダー | 毎日指定時間に「今日の貯金をしよう」通知 |
| 推しの誕生日通知 | 誕生日当日にお知らせ |
| イベント前日通知 | 登録済みイベントの前日にリマインド |
| 貯金目標達成通知 | 目標金額到達時に通知 |

---

## 8. データ管理

### 8.1 ローカルストレージ
- デフォルトはデバイス内にデータ保存（オフライン動作対応）
- 使用ライブラリ：`sqflite`（SQLite）または `Hive`（NoSQL）

### 8.2 バックアップ・エクスポート
- CSV 形式でデータエクスポート（支出履歴）
- JSON 形式での全データバックアップ
- 将来的にクラウド同期（Google Drive / iCloud）への対応を検討

## 9. 画面構成（画面一覧）

```
アプリ起動
└── スプラッシュ画面
    └── ホーム（ダッシュボード）
        ├── 推し一覧

        │   │   ├── イベント一覧 / 詳細 / 登録

        │   │   └── 貯金一覧 / 詳細 / 登録

        ├── 支出サマリー
        │   ├── 月次サマリー
        │   └── 年次サマリー
        └── 設定
            ├── 通知設定
            ├── データエクスポート
            └── アプリ情報
```

## 10. 非機能要件

| 項目 | 内容 |
|---|---|
| 対応 OS | iOS 15.0 以上 / Android 8.0（API 26）以上 |
| オフライン対応 | 全機能オフラインで動作 |
| パフォーマンス | 画面遷移 300ms 以内 |
| データ保護 | 個人データはデバイス内のみに保存（外部送信なし） |
| アクセシビリティ | フォントサイズ変更対応、ダークモード対応 |
