# 推し活管理アプリ 仕様書

**バージョン:** 1.0.0
**作成日:** 2026年7月2日
**プラットフォーム:** Web（主軸）/ iOS / Android（クロスプラットフォーム）

---

## 1. アプリ概要

### 1.1 アプリ名（仮）

**Oshi Log（推しログ）**

### 1.2 コンセプト

推しに関するすべての活動（イベント・グッズ購入・支出・貯金）をひとつのアプリで一元管理する。  
「今月いくら使ったか」「いくら貯められたか」を一目で把握できる、推し活専用の活動記録 × 家計簿アプリ。

### 1.3 ターゲットユーザー

- アイドル・俳優・VTuber・アニメキャラクターなどの推しを持つユーザー
- グッズ購入やイベント参加を記録・管理したいユーザー
- 推しへの支出を把握・コントロールしたいユーザー
- 推しのために計画的に貯金をしたいユーザー

---

## 2. 機能一覧

| # | 機能 | 概要 |
|---|------|------|
| 1 | 推し管理 | 推しの基本情報を登録・管理する |
| 2 | イベント管理 | 参加したイベントの情報を記録・閲覧する |
| 3 | グッズ・購入履歴管理 | 購入したグッズの履歴を記録・閲覧する |
| 4 | 支出サマリー | 月次・年次の支出を集計・グラフ表示する |
| 5 | 推し貯金 | 日々の貯金を記録し、月ごとの貯金額を可視化する |
| 6 | 通知 | 貯金リマインダー・誕生日・イベントを通知する |

---

## 3. 推し管理

### 3.1 推し登録

- グループ単位・個人単位どちらでも登録可能
- グループ登録時、所属メンバーを個別に追加できる
- 各推しに設定できる情報：

| 項目 | 内容 |
|------|------|
| 名前 | テキスト入力 |
| アイコン画像 | ギャラリーから選択 または カメラ撮影 |
| カバーカラー | テーマカラーをカラーピッカーで選択 |
| 誕生日 | カレンダーから選択 |
| カテゴリ | アイドル / 俳優 / VTuber / アニメ / その他 |
| メモ | 自由記述 |

### 3.2 推し一覧

- 登録した推しをカード形式で一覧表示
- タップで推しごとの詳細ページへ遷移
- 並び替え：登録順 / 名前順 / 最近閲覧順

---

## 4. イベント管理

### 4.1 イベント記録

各推しに紐づいてイベントを登録できる。

| 項目 | 内容 |
|------|------|
| イベント名 | テキスト入力 |
| 日付 | カレンダーから選択 |
| 場所・会場 | テキスト入力 |
| カテゴリ | コンサート / 舞台 / 握手会 / 配信 / その他 |
| 支出金額（合計） | 数値入力（円） |
| 内訳 | チケット代・交通費・宿泊費・その他（任意） |
| 写真 | 複数枚添付可能 |
| メモ | 自由記述 |

### 4.2 イベント一覧・閲覧

- 時系列（新しい順 / 古い順）で一覧表示
- 月でフィルタリング可能
- カレンダービューで参加日を可視化
- タップで詳細ページへ遷移・編集・削除が可能

---

## 5. グッズ・購入履歴管理

### 5.1 グッズ登録

各推しに紐づいてグッズの購入情報を登録できる。

| 項目 | 内容 |
|------|------|
| グッズ名 | テキスト入力 |
| 購入日 | カレンダーから選択 |
| カテゴリ | CD / Blu-ray / 写真集 / アパレル / アクセサリー / 雑貨 / その他 |
| 金額 | 数値入力（円） |
| 購入場所 | テキスト（店舗名 / EC サイト名） |
| 数量 | 数値入力 |
| 写真 | 1〜複数枚添付可能 |
| メモ | 自由記述 |

### 5.2 グッズ一覧・閲覧

- グリッド / リスト 表示切り替え
- カテゴリ・月・期間でフィルタリング
- タップで詳細ページへ遷移・編集・削除が可能

---

## 6. 支出サマリー（ダッシュボード）

### 6.1 月次サマリー

- 当月の総支出額を大きく表示
- 内訳をカテゴリ別に円グラフ / 棒グラフで表示
  - イベント費（チケット・交通費・宿泊費など）
  - グッズ費
  - その他
- 推しごとの支出比較（複数推し登録時）

### 6.2 推しごとサマリー

- 選択した推しへの累計投資額
- 月別推移グラフ（折れ線グラフ）
- イベント参加回数・グッズ購入点数の統計

### 6.3 年次サマリー

- 年間の支出推移を月ごとに棒グラフで表示
- 年間総支出額

---

## 7. 推し貯金機能

推しの誕生日や記念日などを起点に、小額を毎日積み立てる機能。  
月ごとにいくら貯められたかを可視化することで、貯金のモチベーションを維持する。

### 7.1 貯金プランの作成

| 項目 | 内容 |
|------|------|
| 貯金名 | 例：「○○ちゃん誕生日貯金」 |
| 紐づく推し | 登録済みの推しから選択 |
| 開始日 | カレンダーから選択（誕生日・記念日など） |
| 目標日（任意） | 目標達成希望日 |
| 目標金額（任意） | 達成したい総額 |
| 1日あたりの基本貯金額 | 数値入力（円） |

### 7.2 毎日の貯金記録

- 「今日も貯金した！」ボタンでチェックイン
- 任意で金額を変更して記録可能（その日だけ増額・減額）
- 連続貯金日数（ストリーク）を表示

### 7.3 貯金の可視化

- **月ごとの貯金合計額**を表示
- 累計貯金額と目標金額の**進捗バー**
- カレンダーヒートマップで貯金した日を可視化
- 目標達成時に達成通知 / お祝いアニメーション

### 7.4 複数の貯金プランを並行管理

- 推しごとに複数の貯金プランを作成可能
- 一覧で各プランの進捗を確認

## 8. 通知機能

| 通知種別 | 内容 | Web での扱い |
|----------|------|-------------|
| 貯金リマインダー | 毎日指定時間に「今日の貯金をしよう」通知 | 画面内バナーで代替 |
| 推しの誕生日通知 | 誕生日当日にお知らせ | 画面内バナーで代替 |
| イベント前日通知 | 登録済みイベントの前日にリマインド | 画面内バナーで代替 |
| 貯金目標達成通知 | 目標金額到達時に通知 | 画面内バナーで代替 |

> **Web 制約:** スケジュール通知・繰り返し通知はブラウザ API の制限により非対応。`kIsWeb` で判定し、Web では画面内バナー通知で代替する。

## 9. データ管理

### 9.1 ローカルストレージ

- デバイス内にデータ保存（オフライン動作対応）
- 個人データは外部サーバーへ送信しない
- **DB ライブラリ:** `drift`（全Platform対応 SQLite ORM）
  - iOS / Android: SQLite ファイル
  - Web: drift/wasm バックエンド（IndexedDB に永続化）

### 9.2 バックアップ・エクスポート

- CSV 形式でデータエクスポート（支出履歴・貯金履歴）
- JSON 形式での全データバックアップ
- 将来的にクラウド同期（Google Drive / iCloud）への対応を検討

    └── ホーム（ダッシュボード）

        │   └── 年次サマリー

## 11. 非機能要件

| 項目 | 内容 |
|------|------|
| 対応環境 | Web（Chrome / Edge / Safari）/ iOS 15.0 以上 / Android 8.0（API 26）以上 |
| オフライン対応 | 全機能オフラインで動作（Web は IndexedDB に永続化） |
| パフォーマンス | 画面遷移 300ms 以内 |
| データ保護 | 個人データはデバイス内のみに保存（外部送信なし） |
| アクセシビリティ | フォントサイズ変更対応、ダークモード対応 |
